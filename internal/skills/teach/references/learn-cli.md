# Learn CLI — Reference

Command reference for the `learn` CLI. Loaded when the teach skill needs the
mechanism; the teach skill decides _when_ and _why_ to run each.

## Workspace management

```bash
learn init "<name>"              # Create a new learning workspace
learn init "<name>" --cwd        # Create in current directory
learn init "<name>" --dir <path> # Create at a custom path
learn init "<name>" --topic "<friendly title>"  # Override the display title

learn workspace list             # List all workspaces
learn workspace open "<name>"     # Open workspace, show path
learn workspace open "<name>" --open  # Open in file manager
learn workspace stats            # Show learning statistics

learn mission   -w "<name>"        # Show mission
learn mission   -w "<name>" --edit # Edit mission in $EDITOR
learn resources -w "<name>"        # Show resources
learn resources -w "<name>" --edit # Edit resources
learn glossary  -w "<name>"        # Show glossary
learn glossary  -w "<name>" --edit # Edit glossary
```

## Lessons

```bash
learn lesson create "<title>" -w "<name>"              # Create lesson scaffold
learn lesson create "<title>" -w "<name>" --open       # Create and open
learn lesson create "<title>" -w "<name>" --body-file <path>  # Content from file (verbatim, no stub)
learn lesson list  -w "<name>"                          # List lessons
learn lesson list  -w "<name>" --search "<q>"           # Search lessons
```

## Learning records

```bash
learn record add "<title>" -w "<name>"                  # Add a learning record
learn record add "<title>" -w "<name>" --summary "..."  # With a summary
learn record add "<title>" -w "<name>" --body-file <path>  # Body from file (verbatim)
learn record list -w "<name>"                           # List records
learn record list -w "<name>" --search "<q>"            # Search records
```

## References

```bash
learn reference create "<title>" -w "<name>"            # Create a reference doc scaffold
learn reference create "<title>" -w "<name>" --body-file <path>  # Content from file (verbatim)
learn reference list -w "<name>"                        # List references
learn reference list -w "<name>" --search "<q>"         # Search references
```

## Dashboard

```bash
learn start              # Start the read-only web UI (default :8080)
learn start --port 9090  # Custom port
learn start --no-open    # Don't auto-open the browser
```

## Global flags

```bash
--json   # Machine-readable JSON output (most commands)
--db     # Custom database path (default: ~/.learn-tool/learn.db)
```

## File naming (generated by the CLI)

- Lessons:    `0001-dash-case-name.html` (4-digit zero-padded sequence)
- Records:    `0001-dash-case-title.md`
- References: descriptive, e.g. `sql-syntax-reference.html`

The sequence numbers are assigned by the CLI on creation — don't hand-number.

## Workspace layout

```
<name>/
├── MISSION.md            # Why — reason the user is learning
├── RESOURCES.md          # Curated knowledge sources and communities
├── GLOSSARY.md           # Canonical terminology (built over time)
├── NOTES.md              # User preferences and working notes
├── lessons/              # Self-contained lesson HTML files
├── learning-records/     # ADR-style records of what was learned
├── reference/            # Cheat sheets and reference documents
└── assets/               # Reusable components (style.css, etc.)
```

The web dashboard is read-only — all mutations go through the CLI.
